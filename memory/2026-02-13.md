## Code Security Policy

**WICHTIG:** Code muss immer von Linus reviewed werden bevor er auf dem Server ausgeführt wird.

- **Sandbox:** Code kann in der Sandbox ausgeführt werden — das ist OK
- **Server/Production:** Code muss zuerst von Linus reviewed und approved werden
- **GitHub:** Code kann auf GitHub gepushed werden (Review passiert vor Deploy)

Diese Policy gilt für alle Agents (coder, devops, etc.).

## Sandbox Issue identifiziert

- Dockerfile hat `git` und `gh` installiert
- Aber: `sandbox.network: "openclaw-isolated"` blockiert externe Verbindungen
- Lösung: `sandbox.mode: "off"` für coder (wie bei devops/main) oder network ändern
